# 应届生搜索页自动投递助手

一个面向 `yingjiesheng.com` / `q.yingjiesheng.com` 搜索结果页、职位推荐页和实习页的 Tampermonkey 自动投递脚本。

适用于用户已经手动完成登录、搜索和筛选的场景。脚本会在当前岗位列表页中自动识别可投递岗位，优先处理站内直投职位，并按规则继续翻页。

## 项目简介

- 适用页面：`yingjiesheng.com` / `q.yingjiesheng.com` 的职位搜索结果页、职位推荐页、实习页
- 脚本定位：岗位列表页自动投递辅助工具，不替代用户登录和搜索筛选操作
- 默认策略：仅处理站内直投岗位，自动跳过外链岗位
- 核心能力：自动投递、自动翻页、发布时间过滤、多关键词匹配模式切换、面板日志和本地设置记忆

## 适用页面

脚本当前匹配以下页面：

- `https://q.yingjiesheng.com/jobs/search/*`
- `https://www.yingjiesheng.com/jobs/search/*`
- `https://q.yingjiesheng.com/pc/searchintention*`
- `https://q.yingjiesheng.com/pc/searchintern*`

请在职位搜索结果页、职位推荐页或实习页使用，不要在职位详情页、首页或其他页面直接启动。

## 核心功能

- 自动识别当前页可投递岗位
- 自动逐个点击 `立即申请`
- 自动处理申请确认弹窗
- 自动识别投递成功提示
- 自动翻页继续投递
- 自动检测“今日投递太多”并停止
- 开始前预检查当前页候选情况
- 支持暂停 / 继续
- 支持成功数 / 失败数 / 跳过数 / 当前页数实时统计
- 日志仅保留最近 `50` 条
- 面板支持拖动到任意位置
- 面板设置和面板位置会保存到本地，刷新页面后自动恢复

## 默认规则

当前脚本内置以下过滤逻辑：

- 仅投递站内直投岗位
- 静默跳过外链岗位
- 可选过滤发布时间超过指定天数的岗位
- 可选按多个关键字进行岗位匹配，并支持两种匹配模式

- `仅投站内直投` 和 `外链静默跳过` 属于脚本内部固定策略，不在界面中暴露开关
- `发布时间过滤` 和 `多关键词匹配` 可以在面板中由用户自行开启或关闭

## 使用说明

1. 打开应届生职位搜索结果页、职位推荐页或实习页
2. 手动完成登录
3. 手动设置职位关键词、城市、学历、工作性质等搜索筛选条件
4. 确认页面已经显示职位列表
5. 观察页面右侧出现 `应届生自动投递` 控制面板
6. 按需要配置过滤项
7. 点击 `开始`，先查看预检查结果
8. 确认预计投递数量符合预期后等待脚本自动执行

## 面板说明

### 按钮

- `开始`：启动自动投递
- `暂停`：暂停当前自动投递；再次点击会继续
- `停止`：手动停止当前自动投递流程
- `清空记录`：清空本轮日志和已处理记录

### 实时统计

- `成功`：本轮已成功投递的岗位数
- `失败`：本轮已尝试但未成功投递的岗位数
- `跳过`：本轮因外链等原因被跳过的岗位数
- `当前页`：当前正在处理的页码

### 过滤项

- `过滤发布时间超过 N 天`：开启后读取职位详情页发布时间，超过设定天数的岗位会被过滤
- `启用关键字匹配`：开启后，脚本会按你输入的多个关键字筛选岗位
- `匹配模式`：
  - `命中任意一个`：岗位文本命中任意一个关键字就会保留
  - `必须命中全部`：岗位文本需要同时命中所有关键字才会保留
  - 选择 `必须命中全部` 时，筛选会明显更严格，候选岗位数量可能大幅减少
- `关键词输入框`：支持输入多个关键字，可用空格、逗号、顿号、分号或竖线分隔；留空时默认读取当前搜索页关键词

## 预检查说明

点击 `开始` 后，脚本会先输出一组预检查日志，例如：

- 当前页职位卡片数
- 当前页可申请岗位数
- 站内直投岗位数
- 按当前过滤条件预计会投递的岗位数

如果预计投递数量与预期不一致，建议先不要继续大范围使用，优先检查：

- 是否在正确的岗位列表页
- 是否开启了过严的关键字匹配
- 是否开启了发布时间过滤

## 日志说明

日志会显示在控制面板底部，格式尽量简洁，常见内容包括：

- `脚本已加载，等待开始`
- `开始自动投递`
- `当前页找到 X 个待投递职位`
- `投递成功: 岗位名`
- `投递失败: 岗位名`
- `当前页未找到可投递职位，准备尝试下一页`
- `已点击下一页`
- `下一页加载完成`
- `未找到下一页按钮，自动投递结束`
- `点击下一页后列表未变化，自动投递结束`
- `检测到提示：您今日投递太多，休息一下明天再来吧`

日志仅保留最近 `50` 条。

## 运行逻辑

脚本的大致处理顺序如下：

1. 收集当前页所有可见职位卡片
2. 找出卡片中的 `立即申请` 按钮
3. 过滤非站内直投岗位
4. 根据用户设置执行发布时间过滤和多关键词匹配
5. 逐个点击申请
6. 处理确认弹窗
7. 判断是否投递成功
8. 当前页结束后自动翻到下一页

## 使用建议

- 请先手动登录，再启动脚本
- 请确保当前页面是搜索结果页、职位推荐页或实习页
- 建议先用较小范围关键词做测试
- 如果页面结构发生变化，按钮选择器或弹窗选择器可能需要重新适配
- 某些岗位虽然按钮文本为 `立即申请`，但如果属于外链投递，脚本会自动跳过
- 如果网站触发每日投递上限，脚本会自动停止

## 已知限制

- 脚本依赖当前页面结构和类名，网站改版后可能失效
- 发布时间过滤依赖职位详情页中存在可识别的发布时间文本
- 关键字匹配基于页面文本包含关系，不是语义理解
- 目前没有做云端同步，也不会跨浏览器同步设置
